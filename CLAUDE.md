# CarbonTrail — Agent Instructions

## What this is
Climate spending intelligence platform for Canadian public funding.
Cross-references contracts, grants, and lobbying data to surface accountability patterns.

## Architecture
- `backend/` — Python FastAPI + DuckDB (embedded OLAP)
- `frontend/` — Next.js 16 + TypeScript + Tailwind + shadcn/ui
- `backend/etl/ingest.py` — ETL pipeline: CSV → DuckDB (~67s, 5.2M records)
- `backend/data/carbontrail.duckdb` — 747MB database (NOT in git, regenerated by ETL)

## Data sources (all in /data/opendata/canada/)
- PSPC contracts (1.26M records, 258K climate-relevant)
- Federal grants (1.28M records, 113K climate-relevant)
- Lobbyist registry (127K registrations, 70K climate-related)
- T3010 charities (83K) + directors (568K)

## Running locally
```bash
# ETL (one-time)
source /data/projects/vigil/.venv/bin/activate
cd backend && python etl/ingest.py

# Backend
uvicorn backend.app.main:app --port 8902

# Frontend
cd frontend && npm install && PORT=3100 npm run dev
```

## Key design decisions
- DuckDB for embedded OLAP (no external DB needed)
- Climate tagging via LIKE patterns on owner_org_title + description fields
- Lobby-funding loops: cross-match lobbyist_registrations ↔ grants/contracts by org name
- Greenwash signals: subject code analysis (SMT-13=Environment, SMT-10=Energy, SMT-25=NatRes)
- Pro-climate messaging throughout UI — transparency protects climate budgets
