# CarbonTrail — Agent Instructions

## What this is
Climate spending intelligence platform. Cross-references Canadian contracts, grants, and lobbying data for $321B+ in climate-related public funding.

## Architecture
- `backend/` — Python FastAPI + DuckDB, 13 routers, 35 API endpoints
- `frontend/` — Next.js 16, 18 pages, Tailwind + shadcn + Recharts + d3-sankey + react-force-graph-2d
- `backend/etl/ingest.py` — ETL: 5.2M records → DuckDB in ~67s
- `backend/data/carbontrail.duckdb` — 747MB (NOT in git, regenerated by ETL)

## Key design decisions
- DuckDB for embedded OLAP (no external DB needed)
- Climate tagging: LIKE patterns on owner_org_title + description fields
- Lobby loops: cross-match lobbyist_registrations ↔ grants/contracts by org name
- Greenwash: subject code analysis (SMT-13=Environment, SMT-10=Energy, SMT-25=NatRes)
- Pro-climate messaging throughout UI
- Next.js API proxy (rewrites /api/* → backend:8902)

## Running
```bash
# Backend
source /path/to/venv/bin/activate  # needs duckdb, fastapi, uvicorn, boto3
cd backend && uvicorn app.main:app --port 8902

# Frontend  
cd frontend && npm install && npm run dev  # port 3000, proxies /api to 8902
```
